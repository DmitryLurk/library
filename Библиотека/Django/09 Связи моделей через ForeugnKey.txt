Для создания связей между моделями (что по сути является созданием связи между таблицами в БД) используется внешний ключ (ForeignKey)

пример: есть таблица с категориями и основная таблица с информацией.

мы прописываем FK в таблицу с основной информацией вида

ForeignKey(<cckrf на первичную модель>, on_delete=<ограничение при удалении>

cat = models.ForeignKey('Category', on_delete=models.PROTECT)

on_delete может быть
models.CASCADE - при удалении записи из первичной удаляется и из основной
models.PROTECT - запись из первичной не удаляется если используется в основной
models.SET_NULL - при удалении первичнки в основной в поле FK пишет NULL
models.SET_DEFAULT - то же что и null только в модели где прописываем FK должно быть определено значение по умолчанию
models.SET() - устанавливает пользовательское занчение
models.DO_NOTHING - в основной ничего не происходит

Здесь первичной таблицей будет таблица меньшая по обьему 'Категории' а вторичной будет основная таблица с инфой

Первичную модель определяем как
class Category(models.Model):
    name = models.CharField(max_length=100, db_index=True)

    def __str__(self) -> str:
        return self.name
