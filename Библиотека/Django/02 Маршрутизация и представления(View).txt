Для отображения информации используется паттерн MTV (Models, Nemplates, Views)
Когда поступает запрос от пользователя сперва все ищется в URLs 
ссылка на view которая в свою очередь собирает шаблоны из templates и данные из model и отображает их пользователю
----------------------------------------------------------------------------
Представления прописываются в приложении в файле views.py
Они могут быть описаны как функцией так и классом
def index(request):				Здесь будет наследоваться класс request
    return HttpResponse("Страница приложения")	а отображаться будет либо через HttpResponse либо через render
----------------------------------------------------------------------------
После этого нужно прописать ссылку на это представление в файле urls.py
в urlpatterns=[]
path('НАЗВАНИЕПРЕФИКСА/', НАЗВАНИЕФУНКЦИИПРЕДСТАВЛЕНИЯ)	(функцию необходимо импортировать)
САЙТ.ру/названиепрефикса

При этом главная столица должна быть прописана с пустым префиксом вида path('', НАЗВАНИЕФУНКЦИИПРЕДСТАВЛЕНИЯ)
----------------------------------------------------------------------------
Желательно в urls импортировать из приложений не штучно а оптом с помощью функции include
path("", include("НАЗВАНИЕПРИЛОЖЕНИЯ.urls"))		Файл urls необходимо создать в приложении

В файле urls необходимо импортировать path
from django.urls import path
from .views import *		импортировать .из текущего каталога views 

urlpatterns = [
    path("", НАЗВАНИЕФУНКЦИИ, name="home"),		И здесь уже прописываем все маршруты конкретно этого приложения
]
Примечание: при импорте через include путь в адресной строке всегда будет выглядеть САЙТ.ру/НАЗВАНИЕПРИЛОЖЕНИЯ/НАЗВАНИЕПРЕФИКСАФУНКЦИИ
----------------------------------------------------------------------------
